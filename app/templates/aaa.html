AAA Testing

What is AAA Testing?

AAA testing is a pattern/structure used to write good unit tests. It stands for Arrange-Act-Assert.

Arrange

Arrange set up the test case like if the test requires any objects, databases, etc. set up or if the test needs to log into an app first. It involves any operations that need to be handled at the start of the test (such as creating a variable). The Calculator program has a @pytest.fixture which clears the calculator history to set up each test. The history test must be cleared before each test is run so that the history count for each test is accurate. The fixture is applied to every function that uses it as a parameter and is declared at the beginning of the unit tests.

Act

Act involves the main thing being tested. It involves calling a function or method and is focused on the target behavior. In the unit tests, the Act step is demonstrated when methods/functions such as Calculator.add_numbers() or Calculator.history_calculations_count() are called in lines such as "assert Calculator.add_numbers(1, 2) == 3". add_numbers and history_calculations_count are the functions being tested and are the target behavior of the calculator programs.

Assert

While the Act steps anticipate and create a response, the Assert steps verify the accuracy of that response. Assert can check numeric or string values and determines if the test passes or fails. It is essentially a statement asserting what the response should be and verifies if the test actually responds in that way. In the unit tests of the Calculator program, there are multiple assert lines which test the function. When writing the program, you assert what the result of the function should be and the unit test checks if your assertion matches what the program actually returns. If your assertion and the program's return matches then the test passes.

Why Is It Important?

Unit testing is important because it ensures all parts of the program are correct and run correctly, increases the efficiency of code improvement and maintenance, and increases the quality of the code. It separates the unit test to ensure it is focused, efficient, and arranged properly, and makes the test much more understandable and manageable.
