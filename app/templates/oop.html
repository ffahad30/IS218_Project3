Calculator Program Explained:

OOP

Object Oriented Programming Principles

What Are The Four Principles?

Encapsulation, Abstraction, Inheritance, Polymorphism

Encapsulation

With encapsulation, a program has a few different objects which communicate with each other. It puts data and the methods that work on that data together in one place. The Calculator program demonstrates encapsulation because all of the calculator's methods and variables, like all the operation and history methods are in one class which is the Calculator class. They are all encapsulated and work together in the Calculator class.

Inheritance

In inheritance, one class inherits properties from another class which allows for reusability of a code so you don't have to repeat code anywhere. In the program, all the calculator's mathematical operations inherit value_a and value_b from the Calculation class so that they do not constantly have to be initialized and that code does not have to be repeated anywhere else in the program.

Polymorphism

In Polymorphism, the same function name is used for different types. For example, len() can be used on both a string and a list in the same program. It can be used to define methods in the child class that have the same name as the methods in the parent class and the child class can inherit the method from the parent class. In the program, get_result is used in the Addition, Subtraction, Multiplication, and Division classes and it is the same function name but it is used for all the different operations within the same program.

Abstraction

In abstraction, the user knows what they are doing but not how the work is being done due to abstraction which hides the internal implementations of a process or method from the user. The user ignores the details of a program and uses it as a whole rather than thinking about or knowing how each part of the program works. The Calculation and Calculator classes demonstrate abstraction because the user knows that they are doing mathematical operations but not that the function calculator runs mathematical functions using value_a and value_b, and clears the history, adds results to history, etc each time. An abstract class is a class in which abstract methods are defined. The calculation class is an abstract base class. An abstract method is a method with no implementation.
